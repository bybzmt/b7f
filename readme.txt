设计思路
1.使用现有功能,不重造轮子
2.程序即:输入和输出
3.严类型

设计层次
1.输入层    用于解除环境绑定,输入层总是执行成功且返回固定格式的数据
2.验证层    验证用户输入有效性
3.动作层    进行业务操作,要么成功要么异常
4.输出层    转换数据为输出所需格式

0.路由层    入口即路由
0.控制层    .....
